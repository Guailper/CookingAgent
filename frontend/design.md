# 前端界面设计

## 登录界面

### 1. 设计目标

本次登录界面需要回到参考图那种“单卡片居中”的表达方式，不再使用当前左右分栏的信息型首页布局。

核心目标如下：

- 登录窗口始终处于页面视觉中心，用户打开页面后第一眼只聚焦登录动作。
- 风格轻、柔和、干净，接近参考图中的磨砂白卡片 + 淡蓝灰背景。
- 信息密度较低，只保留登录必须内容，避免当前界面中过多说明性模块干扰视线。
- 保留基础登录能力，同时给注册入口留出明确位置。

### 2. 页面整体结构理解

#### 2.1 整体布局

整个页面只保留一个主视觉区域：

- 最外层是全屏背景容器，宽高覆盖整个视口。
- 背景采用非常淡的冷色系渐变，主色倾向于白色、浅灰蓝、极浅雾蓝。
- 在背景中可以加入非常弱的模糊光斑或弧形装饰，但只能作为气氛，不应该抢走登录卡片的注意力。
- 登录卡片水平、垂直双居中，移动端仍然保持居中，但卡片宽度随屏幕自适应缩小。

#### 2.2 视觉重心

视觉重心应完全落在中间登录卡片：

- 页面中不再出现左侧介绍区、统计区、大段说明文案。
- 不再出现偏“产品宣传页”的信息模块。
- 用户进入页面后，应立即识别出“这是登录页”。

### 3. 登录卡片设计理解

#### 3.1 卡片外观

登录卡片是整个页面唯一的主体容器，外观特征参考上传图片：

- 卡片整体为竖向长方形，四角圆润，圆角偏大。
- 卡片背景为接近纯白的浅色，视觉上带一点柔和阴影。
- 卡片宽度在桌面端建议控制在 `380px - 440px` 之间。
- 卡片内部上下留白要充足，让界面显得呼吸感很强。
- 阴影不能太重，应该是软阴影，形成轻微悬浮感。

#### 3.2 内容分区

卡片内部从上到下分成 5 个区域：

1. 标题区
2. 表单区
3. 主按钮区
4. 第三方登录区
5. 注册引导区

各区之间有明显但柔和的垂直节奏。

### 4. 标题区理解

标题区位于卡片顶部居中：

- 主标题使用 `Welcome Back`，居中显示。
- 副标题为“Please enter your details to sign in”，颜色比主标题浅一级。
- 主标题要明显粗于副标题，形成强弱层次。
- 标题和表单之间需要留出明显间距。

视觉感觉应该是：

- 主标题明确
- 副标题克制
- 不做过多品牌展示

也就是说，这一版登录页是“任务导向型”而不是“品牌导向型”。

### 5. 表单区理解

#### 5.1 表单字段

登录区只包含两个输入项：

- Email Address
- Password

这两个字段遵循参考图中的排布方式：

- 字段标签在输入框上方。
- 标签字号较小，颜色偏中灰。
- 输入框本体为圆角胶囊形，不是硬直角矩形。

#### 5.2 输入框样式

输入框应具有以下特征：

- 高度适中，偏圆润。
- 背景色为浅灰蓝色块，而不是纯白描边输入框。
- 边框感很弱，主要靠底色与页面区分。
- 占满卡片内部可用宽度。
- placeholder 颜色略浅，输入文字颜色比 placeholder 更深。

#### 5.3 Password 行特殊处理

Password 标签行需要左右分布：

- 左侧是 `Password`
- 右侧是 `Forgot Password?`

其中 `Forgot Password?` 是文字按钮：

- 颜色略偏蓝
- 字重略高于普通说明文字
- hover 时可以增加颜色对比或轻微透明度变化

### 6. 主按钮区理解

主按钮是卡片中最强的操作锚点：

- 文案使用 `Sign In`
- 按钮宽度占满表单区域
- 按钮为深蓝色实底
- 文字为白色
- 按钮圆角同样偏大，和输入框风格统一
- 按钮应有轻微阴影，让它比输入框更“可点击”

按钮视觉上要满足两个要求：

- 一眼知道这是主操作
- 不显得过于厚重或企业后台化

### 7. 第三方登录区理解

主按钮下方是第三方登录区域，结构参考图片：

- 使用一条横向分隔结构，中间放 `OR CONTINUE WITH`
- 左右各有一小段浅色分割线
- 这一行的存在感要弱于主按钮，不可过于突出

第三方按钮包含两个：

- Google
- GitHub

按钮特征：

- 两个按钮并排排列
- 背景为很浅的灰蓝白色
- 圆角较大
- 按钮内部左侧为平台图标，右侧为平台名称
- 按钮视觉权重低于主登录按钮

### 8. 注册引导区理解

卡片最底部保留一句注册引导：

- 文案结构为 `Don't have an account? Register`
- 前半句是普通说明文字
- `Register` 是强调文本按钮
- 整体居中显示

- 点击 `Register` 后再切换到注册界面
- 当前这一版先优先完成登录页的视觉还原

### 9. 颜色与气质理解

这一版参考图的气质不是科技炫光型，也不是深色后台型，而是：

- 柔和
- 安静
- 可信
- 现代
- 极简

推荐颜色倾向：

- 页面背景：`#f3f6fb` 附近的浅蓝白
- 卡片背景：`#ffffff`
- 输入框背景：`#eef1f7` 或接近的浅灰蓝
- 主文字：`#2d3340` 附近
- 次文字：`#8b93a3` 附近
- 主按钮：`#355786` 或同类深蓝色

整体不能出现高饱和、强对比、过度渐变。

### 10. 间距与尺寸理解

这一版成败很大程度取决于节奏而不是装饰，因此实现时需要重视间距：

- 卡片上下内边距要大于左右内边距
- 标题区与表单区之间留白明显
- 每个表单项之间保持稳定垂直间距
- 主按钮与第三方登录区之间要有一段呼吸感
- 第三方登录区与底部注册引导之间也要保留足够空间

如果间距过紧，页面会立刻失去参考图那种轻盈感。

### 11. 响应式理解

虽然参考图是桌面截图，但实现时需要考虑移动端：

- 小屏幕下卡片宽度改为接近屏宽减安全边距
- 卡片仍然保持居中
- 第三方登录按钮可以继续双列，但空间不足时允许适当缩小间距
- 文案不换风格，只做尺寸和边距收缩

## 主界面补充设计同步

本轮主界面聊天页补充以下布局约束：

- 左侧最近会话列表中的每条记录只保留单行标题，标题过长时省略显示，列表区域按侧栏剩余高度滚动
- 聊天消息气泡改为按内容宽度收缩显示，并把消息时间放到该条消息右上角的小信息区中
- 左侧边栏需要以视口高度为边界，顶部品牌区和底部用户区固定在栏内，只有中间最近会话列表允许纵向滚动
- 聊天主区域需要以视口高度为边界，顶部工具栏与底部输入框始终保持可见，只有中间消息内容允许随着消息增加而内部滚动
